1 Authentification — JWT (JSON Web Token)
• Algorithme : HS256 (HMAC-SHA256)
• Expiration : 24 heures · token invalidé automatiquement
• Clé secrète : 64 caractères hexadécimaux générée avec secrets.token_hex(32)
• Transmission : Header Authorization: Bearer … uniquement
• Exception streaming vidéo : ?token= accepté sur /api/videos/stream/ (balise HTML <video>)
• Erreurs distinctes : Token expiré vs Token invalide

2 Autorisation — RBAC (Contrôle d'accès par rôles)
• 2 rôles : admin (accès total) et user (accès restreint aux propres ressources)
• Décorateur @token_required → vérifie JWT sur toutes les routes protégées
• Décorateur @admin_required → réservé gestion utilisateurs (CRUD)
• Isolation données : un user ne voit que SES vidéos et analyses (filtre user_id en BD)
• Suppression / modification : admin ou propriétaire uniquement
• Changement mot de passe : admin ou utilisateur lui-même
@token_required
@admin_required
Filtre user_id MongoDB

3 Anti Brute-Force — Rate Limiting
• Bibliothèque : Flask-Limiter 3.5.0
• Limite login : 10 tentatives/minute et 50 tentatives/heure par IP
• Identifiant : adresse IP source (get_remote_address)
• Réponse : HTTP 429 + message "Trop de tentatives. Réessayez dans quelques minutes."
• Testé en production : bloque à la 11ème tentative

4 Stockage mots de passe — bcrypt
• Algorithme : bcrypt avec salt aléatoire unique par mot de passe
• Cost factor : 12 (résistant aux GPU modernes)
• Jamais stocké en clair : hash bcrypt uniquement en base MongoDB
• Résistant à : rainbow tables, attaques dictionnaire, force brute GPU
• Mot de passe admin initial : généré aléatoirement (secrets.token_urlsafe(16)) si non défini

5 Validation des uploads — Sécurité fichiers
• Whitelist extensions : mp4, avi, mov, mkv, flv, wmv, webm uniquement
• Protection path traversal : werkzeug.utils.secure_filename()
• Taille maximale : 500 MB (MAX_CONTENT_LENGTH)
• Nom fichier sécurisé : préfixé par ObjectId MongoDB (unicité garantie)
• Répertoire isolé : /build/backend/uploads/ (hors du code source)

6 Isolation réseau — CORS + Docker network
• CORS restreint : origines explicites via variable CORS_ORIGINS
• Origines autorisées : localhost:3000 (Docker) et localhost:5173 (Vite dev)
• Réseau Docker privé : amane_nexus_net (bridge isolé)
• MongoDB port 27017 : accessible uniquement entre conteneurs, non exposé publiquement
• Redis port 6379 : idem, réseau interne uniquement
• Variables secrets passées via .env (exclu du git via .gitignore)

7 Traçabilité — Journal d'activité (Audit Log)
• Collection MongoDB : activity_log
• Actions enregistrées : LOGIN, LOGOUT, CREATE_USER, DELETE_USER, UPDATE_USER
• Champs par entrée : user_id, username, action, IP source, méthode HTTP, endpoint, statut, timestamp UTC
• Consultable via page Paramètres (admin uniquement)
• Utilité : détection d'intrusion, forensics, conformité
